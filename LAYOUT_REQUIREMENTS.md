# Dashboard 布局调整需求

## 📐 目标布局结构

根据用户提供的参考布局，需要实现以下结构：

**布局说明：左侧系统菜单 + 右侧4列主内容区域**

```
┌─────────────────────────────────────────────────────────────────────┐
│  Header: MOP 主控制台 | 系统管理员 [已连接] | 刷新 | 退出            │
├──────────┬───────────────────────────────────────────────────────────┤
│          │  ┌──────┬──────┬──────────────┬──────┐                  │
│ 系统菜单 │  │ 用户 │ 创建 │   我的房间    │ 房间 │                  │
│          │  │ 信息 │ 房间 │   (跨2列)     │ 二维码│                  │
│          │  ├──────┼──────┼──────────────┼──────┤                  │
│  (左侧)  │  │ 设备 │ 系统 │              │      │                  │
│          │  │ 管理 │ 统计 │              │      │                  │
│          │  └──────┴──────┴──────────────┴──────┘                  │
│          │  ┌────────────────────────────────────────────────────┐  │
│          │  │  用户管理（仅管理员可见）                            │  │
│          │  │  (全宽，跨4列)                                      │  │
│          │  └────────────────────────────────────────────────────┘  │
│          │  ┌────────────────────────────────────────────────────┐  │
│          │  │  邀请码管理（仅超级管理员可见）                      │  │
│          │  │  (全宽，跨4列)                                      │  │
│          │  └────────────────────────────────────────────────────┘  │
└──────────┴───────────────────────────────────────────────────────────┘
```

**4列布局详细说明：**
- **第1列**：用户信息（行1）、设备管理（行2）
- **第2列**：创建房间（行1）、系统统计（行2）
- **第3-4列**：我的房间（跨2列，跨2行，从行1开始）
- **第4列**：房间二维码（行2右侧，或根据实际布局调整）

**布局分配：**
```
行1: [用户信息] [创建房间] [    我的房间 (跨2列)    ]
行2: [设备管理] [系统统计] [我的房间继续] [房间二维码]
```

## 🎨 布局要求

### 1. 左侧系统菜单栏
- **位置**：页面左侧，固定宽度
- **宽度**：建议 200-250px（桌面端）
- **功能**：
  - 显示系统菜单项
  - 可折叠/展开（可选）
  - 固定定位，内容滚动时保持可见
- **样式**：
  - 白色背景
  - 标题："系统菜单"
  - 菜单项列表（待定义具体菜单项）

### 2. 顶部 Header
- **位置**：页面顶部，全宽
- **内容**：
  - 左侧：🏠 MOP 主控制台
  - 右侧：用户昵称 | 连接状态 | 刷新按钮 | 退出按钮
- **样式**：
  - 紫色到蓝色渐变背景
  - 白色文字
  - 固定定位（可选）

### 3. 主内容区域（4列布局）
- **布局方式**：CSS Grid（4列）
- **网格结构**：
  ```
  第一行（4列）：
  - 第1列：用户信息
  - 第2列：创建房间
  - 第3-4列：我的房间（跨2列）
  - 第4列：房间二维码（与我的房间共享右侧空间，可能需要调整）
  
  或者更合理的布局：
  第一行（4列）：
  - 第1列：用户信息
  - 第2列：创建房间
  - 第3列：我的房间（开始）
  - 第4列：房间二维码
  
  第二行（4列）：
  - 第1列：设备管理
  - 第2列：系统统计
  - 第3列：我的房间（继续，跨行）
  - 第4列：（空或我的房间继续）
  
  第三行（全宽，跨4列）：
  - 用户管理（仅管理员可见）
  
  第四行（全宽，跨4列）：
  - 邀请码管理（仅超级管理员可见）
  ```
  
  **注意**：根据图片描述，"我的房间"卡片是垂直跨越的，可能需要跨2行。

### 4. 卡片样式
- **统一卡片样式**：
  - 白色背景
  - 圆角边框
  - 阴影效果
  - 内边距统一
  - 标题带图标

### 5. 响应式设计
- **桌面端（> 1024px）**：
  - 4列网格布局
  - 左侧菜单栏显示（固定宽度）
  - 卡片正常大小
  - "我的房间"可能跨2列和2行

- **平板端（768px - 1024px）**：
  - 2-3列网格布局（根据屏幕宽度调整）
  - 左侧菜单栏可折叠
  - 卡片适当缩小
  - "我的房间"跨列数减少

- **移动端（< 768px）**：
  - 单列布局
  - 左侧菜单栏隐藏（汉堡菜单）
  - 卡片全宽显示
  - 所有跨列卡片恢复为单列
